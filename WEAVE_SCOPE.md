## What it is
Every night the app auto-assembles what each family member did that day — Jonathan's log,
Helen's note, Aurelia's photo, Rafa's voice clip — into ONE woven page, told as a single story
but shown through each person's own lens (theme/font). Save it as a **video keepsake** to Apple
Photos; across a trip the weaves become "a little book." Nobody assembles it — the app does.
Design source: `design_handoff_family_trips/src/ft2/shared.jsx` → `WEAVE` + `TheWeave` (~L774–900).

## Decisions LOCKED (Jonathan, 2026-06-05)
- **Claude-woven from the start.** The narrative (title + opening/closing framing) is generated by
  Claude via the existing worker — NOT a template. Claude writes only the *connective tissue*; the
  beats stay the family's real words/photos/voice verbatim (no fabrication, never rewrites content).
- **Video keepsake, not a static image.** "An image of a motion artifact is not worthwhile." Render
  the woven page's animation to a portrait **1080×1350 mp4** and hand it to the OS **share sheet** →
  Save Video → Apple Photos. (A PWA canNOT write an iCloud shared album directly — no API. The video
  + share sheet is the honest path; the user drops it in Team RAHJ themselves.)
- **Travel stat = the whole family's distance** (not just Jonathan's "driving"). Real, computed.

## Data reality (VERIFIED — the inputs already exist)
- Memories carry `kind` (**photo** | **voice** | **text**), `authorTraveler`, and `capturedAt`/
  `createdAt` (`app/src/lib/memoryStore.js`). → group a day's memories by (person, day) and render
  each as its beat by kind. **No new capture surfaces needed.** `VoiceRecorder.jsx` is real audio.
- Beat kinds map: photo→frame (caption), voice→clip (waveform+transcript), text→words (quote),
  log→Jonathan's day (see stat below). Avatars/dots via the consolidated `TRAVELER_DOT`.

## The travel stat — REAL road miles (the drive-time tool SHIPPED 2026-06-06)
**Use `lib/driveRoute.fetchRoadRoute(stops).miles`** — real Google Routes road distance via the
worker `POST /route` (content-addressed cache; auto-recomputes on a schedule change). Slice 1 calls
it per day: `fetchRoadRoute(thatDay'sStops)` → real road miles for the family's shared journey,
shown e.g. "Day 2 · 540 mi · 4 stops". Graceful: returns null (→ omit/estimate) when offline.
- The haversine ×1.2 ESTIMATE below is SUPERSEDED by the real tool — keep only as a sanity-check
  ballpark (the live tool returned a plausible real number for the fixture trip in verification):
  jackson-2026 ~2,605 mi · nyc ~346 · volleyball ~277 · total ~3,229 mi (straight-line ×1.2).
- The `/route` tool also returns the road **polyline** (already wired into the maps) and duration,
  so the Weave can show real road geometry too if useful.

## Slices (build order)
1. **The woven page (on-screen)** — group a day's real memories by person → animated braid (beat by
   kind) → new worker `/weave` endpoint where Claude generates title + framing from the day's
   contributions + the computed travel stat. Couples to the worker (Claude) → a worker deploy + token
   cost. This is the experience + the magic.
2. **The video keepsake** — canvas motion renderer of the page → **WebCodecs** `VideoEncoder` → mp4 →
   share sheet → Apple Photos. The flagship artifact. WebCodecs-gated (iOS 16.4+, same gate as the
   photo-importer video path); the actual Save-to-Photos is DEVICE-ONLY, not CI-verifiable.
3. **defer** — nightly auto-generation (worker cron) + "the trip becomes a little book" (compile the
   kept weaves). Each its own increment.
